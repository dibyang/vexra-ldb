# 安全政策

## 支持版本

当前项目仍处于 `0.x` 阶段。默认只对最新主干和最新发布版本接收安全修复。

| 版本 | 支持状态 |
| --- | --- |
| `0.1.x` | 支持 |
| 更早版本 | 不承诺支持 |

## 报告安全问题

请不要在公开 issue 中披露可利用的安全问题。可以通过以下方式私下报告：

- Email: `dib.yang@gmail.com`

报告中请尽量包含：

- 影响版本或提交号。
- 复现步骤或最小复现代码。
- 影响范围，例如数据损坏、越权读取、任意文件写入、拒绝服务或依赖供应链风险。
- 你认为安全的缓解建议。

## 响应流程

1. 维护者确认收到报告。
2. 评估影响范围和修复优先级。
3. 在私下完成修复、测试和发布准备。
4. 发布修复版本或补丁说明。
5. 在适当时间公开披露问题摘要。

## 范围说明

以下问题通常按普通 bug 处理，除非能够造成明确安全影响：

- 仅影响测试代码的失败。
- 需要本地完全可信调用方才能触发的异常。
- 未暴露敏感数据、未破坏持久化数据的普通崩溃。

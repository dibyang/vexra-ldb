# 安全政策

[English](SECURITY.en.md) | 中文

## 支持版本

当前项目仍处于 `0.x` 阶段。默认只对最新主干和最新发布版本接收安全修复。

| 版本 | 支持状态 |
| --- | --- |
| `0.2.x` | 支持 |
| `0.1.x` | 有限支持 |
| 更早版本 | 不承诺支持 |

## 报告安全问题

请不要在公开 issue 中披露可利用的安全问题。可以通过以下方式私下报告：

- Email: `dib.yang@gmail.com`
- GitHub 私有漏洞报告：<https://github.com/dibyang/vexra-ldb/security/advisories/new>

如果私下渠道暂时不可用，可以先创建一个不包含漏洞细节的公开占位 issue，只说明“需要私下沟通安全问题”，不要附复现、攻击载荷、日志或受影响数据。

报告中请尽量包含：

- 影响版本或提交号。
- 复现步骤或最小复现代码。
- 影响范围，例如数据损坏、越权读取、任意文件写入、拒绝服务或依赖供应链风险。
- 你认为安全的缓解建议。

请将以下类型的问题视为安全敏感问题：

- 数据泄露、越权读取或绕过访问控制。
- 任意文件读取、写入、删除或路径穿越。
- 可利用的反序列化、插件加载或依赖供应链风险。
- 可被远程或非可信输入稳定触发的拒绝服务。
- 可导致持久化数据损坏、恢复绕过或备份污染的攻击路径。

## 响应流程

1. 维护者确认收到报告。
2. 评估影响范围和修复优先级。
3. 在私下完成修复、测试和发布准备。
4. 发布修复版本或补丁说明。
5. 在适当时间公开披露问题摘要。

## 范围说明

以下问题通常按普通 bug 处理，除非能够造成明确安全影响：

- 仅影响测试代码的失败。
- 需要本地完全可信调用方才能触发的异常。
- 未暴露敏感数据、未破坏持久化数据的普通崩溃。

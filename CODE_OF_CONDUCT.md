# 行为准则

`vexra-ldb` 希望保持专业、友善、以问题为中心的协作环境。参与 issue、PR、代码评审、文档讨论或其他项目沟通时，请遵守以下约定。

## 我们鼓励

- 尊重不同经验背景的参与者。
- 聚焦事实、代码、测试和设计约束。
- 对风险和分歧给出可复现证据。
- 用清晰、建设性的方式提出改进建议。
- 承认错误并及时修正。

## 我们不接受

- 人身攻击、骚扰、歧视性表达或威胁。
- 泄露他人隐私或未经授权披露安全问题细节。
- 在技术讨论中反复发布无关内容。
- 明知会破坏数据或安全却诱导他人执行的操作。

## 执行方式

维护者可以对违反行为准则的内容采取提醒、编辑、隐藏、关闭讨论、限制参与或上报平台等措施。对于安全问题，请优先遵循 `SECURITY.md` 中的私下报告流程。
